# Building a CTI Program

## 1. Orientation

### What this module covers
The Capstone. How to build a CTI function from scratch. Staffing, tooling, and maturity models.

### Learning Objectives
*   Design a **Team Structure**.
*   Select **Core Tooling**.
*   Measure **Program Maturity**.

---

## 2. Core Content

### Phase 1: Ad-Hoc (The Firefighter)
*   1 person part-time.
*   Reading blogs, forwarding links.
*   No budget.
*   *Goal:* Prove value.

### Phase 2: Defined (The Detective)
*   Dedicated Analyst.
*   Free tools (MISP).
*   Consuming feeds. Basic parsing.
*   *Goal:* Consistency.

### Phase 3: Optimized (The Sentinel)
*   Full Team (Strategic + Operational + Tactical).
*   Paid feeds/Tools.
*   Generating original Intel.
*   *Goal:* Predictive capability.

### 4. Staffing
*   **Skillset:** Critical thinking > Python > Malware Analysis > Writing.
*   **Mix:** Hire diverse backgrounds (Journalism, Military, Tech).

---

## 3. Lab: Pitching the Budget
**Task:** Ask the CISO for $200k for a TIP.
**Pitch:** "Currently, we spend 20 hours/week manually copying IPs. A TIP automates this, saving $50k/year in labor and reducing reaction time from Days to Seconds. It prevents the $5M breach."

---

## 4. Completion Criteria
1.  **Define** CTI Maturity.
2.  **Argue** for "Critical Thinking" as the #1 skill.
