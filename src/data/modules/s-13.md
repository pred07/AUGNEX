# OSINT Tooling (Maltego & SpiderFoot)

## 1. Orientation

### What this module covers
Manual OSINT is slow. Tools automate the collection. **SpiderFoot** handles the broad scan (100+ modules). **Maltego** handles the visual link analysis. This module connects the concepts from s-34 (Link Analysis) specifically to OSINT sources.

### Learning Objectives
*   Configure **SpiderFoot** scans.
*   Visualize data in **Maltego**.
*   Identify the limitations of automated tools.

---

## 2. Core Content

### SpiderFoot
*   **Function:** Automated reconnaissance.
*   **Modules:** Checks Blacklists, DNS, Whois, Social Media, Email leaks, etc.
*   **Output:** Massive CSV/Report. Great for "Broad" recon.

### Maltego Community Edition
*   **Function:** Visual correlation.
*   **Transforms:** "To Email", "To Phone", "To Domain".
*   **Power:** Showing relations. (e.g., These 3 domains share the same Google Analytics ID).

### The Workflow
1.  **Target:** `evil.com`.
2.  **SpiderFoot:** Run scan. Export results.
3.  **Maltego:** Import interesting IPs/Emails. Pivot to find hidden infrastructure.

## 4. Reflection
1.  Why verify automated results? (Tools generate False Positives. SpiderFoot might say "Email Found: `admin@example.com`" which is just a placeholder text on the site, not a real email).
2.  What is "API Rate Limiting"? (Tools use APIs. If you don't have paid keys (e.g., Shodan, BuiltWith), the tool will fail or return limited results).
