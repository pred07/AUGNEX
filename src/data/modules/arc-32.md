# Insider Threat Program

## 1. Orientation

### What this module covers
This module covers **The Enemy Within**. Malicious or Accidental.

### Fit in the Learning Path
Snowden. Manning. Levandowski. The most damaging attacks come from trusted users.

### Learning Objectives
By the end of this module, you should understand:
*   **Behavioral Indicators**.
*   **Privacy vs Security**.
*   **The Exit Interview**.

---

## 2. Core Content

### Indicators
*   **Technical**: Downloading large amounts of data at 3 AM. Accessing files not related to job.
*   **Behavioral**: Disgruntled. Financial distress. Working odd hours.

### The Program
1.  **Baseline**: What is normal?
2.  **Monitor**: UEBA (User and Entity Behavior Analytics).
3.  **Intervene**: HR + Legal + Security.

---

## 3. Guided Practice

### The "Departure" Workflow
**Scenario**: Top Salesperson resigns to join a competitor.
**Action**:
1.  **Immediate**: Revoke Access.
2.  **Forensics**: Review logs for the last 30 days. Did they copy the "Client List"?
3.  **Legal**: If theft found, Legal sends a "Cease and Desist".

---

## 4. Reflection Check

1.  **Trust**: If you treat everyone like a criminal, you create criminals. Be transparent.
2.  **Privileged Users**: Admins are the highest risk. Monitor admin actions (PAM).
3.  **Sabatoge**: It's not just theft. It's logic bombs. Code reviews prevent this.

---

## 5. Completion Criteria

This module is complete when:
1.  You have a **UEBA** strategy.
2.  You collaborate with **HR**.
3.  You have a strictly defined **Offboarding Process**.
