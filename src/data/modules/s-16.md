# MITRE ATT&CK Mapping & Usage

## 1. Orientation

### What this module covers
The **MITRE ATT&CK Framework** is the periodic table of hacker behavior. It is the common language of CTI. You will learn to map reports to T-Codes (e.g., T1003) and use the **ATT&CK Navigator** to visualize coverage.

### Learning Objectives
*   Navigate the **Matrix** (Tactics vs Techniques vs Sub-techniques).
*   Map a text report to **T-Codes**.
*   Create a **Heatmap** in ATT&CK Navigator.

---

## 2. Core Content

### The Structure
*   **Tactics (Columns):** The "Why". (Initial Access, Execution, Persistence).
*   **Techniques (Cells):** The "How". (Phishing, PowerShell, Scheduled Task).
*   **Sub-Techniques:** Specifics. (T1059.001: PowerShell).

### Use Cases
1.  **Mapping Intel:** "They used PowerShell" -> **T1059.001**.
2.  **Gap Analysis:** "We have 0 detections for Exfiltration (TA0010)."
3.  **Adversary Emulation:** "APT29 uses T1090. Simulate that."

### The Navigator
A web-based tool to color-code the matrix.
*   **Blue:** Techniques APT29 uses.
*   **Red:** Techniques we can detect.
*   **Purple:** Overlap (We can detect what they use).
*   **White:** Gaps (They use it, we can't detect it).

## 3. Lab: Mapping a Report
**Source Text:** "The actor sent a spearphishing link (1). User clicked, downloading a VBS script (2). The script created a Registry Run Key (3) for persistence."

**Mapping:**
1.  **T1566.002:** Phishing: Spearphishing Link.
2.  **T1059.005:** Command and Scripting Interpreter: Visual Basic.
3.  **T1547.001:** Boot or Logon Autostart Execution: Registry Run Keys / Startup Folder.

---

## 4. Completion Criteria
1.  **Differentiate** Tactic (Goal) vs Technique (Method).
2.  **Find** the T-Code for "Credential Dumping". (T1003).
