# Executive Reporting

## 1. Orientation

### What this module covers
This module covers translating "Rootkit" into "Risk of Revenue Loss".

### Fit in the Learning Path
The C-Suite controls the budget. You must speak their language.

### Learning Objectives
By the end of this module, you should understand:
*   **Risk Quantification**.
*   **The "So What?" Test**.
*   **Visual Storytelling**.

---

## 2. Core Content

### The Translation
*   *Tech*: "We patched CVE-2023-1234 on 50 servers."
*   *Exec*: "We prevented a potential outage of the e-commerce platform during Black Friday."

### The Report Structure
1.  **Executive Summary**: 1 page max. Red/Yellow/Green status.
2.  **Key Wins**: What did we stop?
3.  **Key Risks**: What keeps us up at night?
4.  **Ask**: What do we need? (Budget/Headcount).

---

## 3. Guided Practice

### Drafting the Monthly Update
**Scenario**: Phishing attempts are up 200%.
**Bad**: "SMTP logs show increased volume."
**Good**: "The company is under active targeting. We need to implement MFA to mitigate this surge."

**Exercise**: Rewrite a technical finding for a CFO.
*   *Finding*: SQL Injection possible on /login.
*   *Rewrite*: The customer database allows unauthorized access, risking GDPR fines.

---

## 4. Reflection Check

1.  **Fear**: Don't just sell FUD (Fear, Uncertainty, Doubt). Sell Solutions.
2.  **Honesty**: "We are currently Red" is better than lying about being Green.
3.  **Brevity**: Executives read headlines. Make them count.

---

## 5. Completion Criteria

This module is complete when:
1.  You can translate **Technical Jargon** to **Business Risk**.
2.  You create a **1-Page Executive Summary**.
3.  You link Security Goals to **Business Goals**.
