# Advanced Search Engine Techniques (Dorking)

## 1. Orientation

### What this module covers
Search engines index 90% of the visible web, but keywords only reach 10%. **Google Dorking** (Advanced Operators) allows you to surgically extract sensitive files, login portals, and configuration errors.

### Learning Objectives
*   Master the `filetype`, `site`, `inurl`, and `intitle` operators.
*   Locate exposed spreadsheets and invoices.
*   Use non-Google engines (Bing, Yandex, Baidu) for regional intel.

---

## 2. Core Content

### The Operators

| Operator | Function | Example |
| :--- | :--- | :--- |
| `site:` | Limit to domain | `site:tesla.com` |
| `filetype:` | Specific extension | `filetype:pdf` or `filetype:xlsx` |
| `inurl:` | Text in the URL | `inurl:admin` or `inurl:login` |
| `intitle:` | Text in page title | `intitle:"index of"` |
| `-` | Exclude term | `site:linkedin.com -site:www.linkedin.com` |

### The "Juicy" Dorks
*   **Finding Passwords:** `filetype:env intext:password`
*   **Finding Network Maps:** `filetype:vsd` (Visio) or `filetype:pcap`
*   **Finding Exposed Indexes:** `intitle:"index of" "parent directory"`

### Beyond Google
*   **Yandex:** Best for facial recognition and Russia/Eastern Europe.
*   **Baidu:** Essential for China.
*   **Bing:** Often indexes code/files that Google DMCA removes.

## 3. Lab: Dorking for Bounty

### Target: Bugcrowd (Simulated)
*   Find all PDF documents on `nasa.gov`: `site:nasa.gov filetype:pdf`
*   Find login portals on `tesla.com`: `site:tesla.com inurl:login`
*   Find exposed configuration files: `site:github.com "aws_access_key_id"`
