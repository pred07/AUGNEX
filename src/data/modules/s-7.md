# Advanced Search Engine Techniques (Dorking)

## 1. Orientation

### What this module covers
Google is a hacking tool. "Google Dorking" (Google Hacking) involves using advanced operators to find security holes, exposed files, and sensitive administrative panels that were accidentally indexed.

### Learning Objectives
*   Master `site:`, `filetype:`, `intitle:`, `inurl:` operators.
*   Locate **exposed configuration files** and **passwords**.
*   Use **Google Hacking Database (GHDB)**.

---

## 2. Core Content

### The Operators
*   `site:example.com` - Limit results to this domain.
*   `filetype:pdf` - Only show PDFs. (Try `docx`, `xlsx`, `conf`, `env`).
*   `intitle:"index of"` - Finds Directory Listings (Open directories).
*   `intext:"password"` - Searches body text.
*   `inurl:admin` - Looks for "admin" in the URL.
*   `cache:example.com` - View Google's cached version (Passive Recon).

### "The Juice" (Common Dorks)
1.  **Finding Login Portals:**
    *   `site:target.com inurl:login`
    *   `site:target.com intitle:"Unknown - Login"` (VPN portals)
2.  **Finding Sensitive Files:**
    *   `site:target.com filetype:xlsx "password"`
    *   `site:target.com filetype:env` (Environment variables often containing API keys).
3.  **Finding Subdomains:**
    *   `site:*.target.com -www` (Show me subdomains that are NOT www).

### The Google Hacking Database (GHDB)
Maintained by Exploit-DB. A repository of thousands of dorks to find:
*   Vulnerable servers.
*   Advisory warnings.
*   Juicy info.

## 3. Lab: Dorking a Bug Bounty Target
*Ethical Warning: Do not use this on unauthorized targets.*

1.  **Target:** `bugcrowd.com` (Authorized scope only).
2.  **Query:** `site:bugcrowd.com filetype:pdf`
3.  **Analyze:** Read public reports, whitepapers.
4.  **Query:** `site:bugcrowd.com -www`
5.  **Analyze:** Discover `docs.bugcrowd.com`, `tracker.bugcrowd.com`.
